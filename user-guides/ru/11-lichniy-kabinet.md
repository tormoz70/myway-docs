# Личный кабинет

Персональные функции теперь **встроены в основное меню** (группа **«Моё»**), а не спрятаны под отдельным пунктом «Личный кабинет». Прямой переход на `/manage/me` перенаправляет на «Главную».

![Личная «Главная» ученика](../assets/me-home.png)
*Рис. Личная «Главная» ученика: ближайшее занятие, пропуск, кнопка прохода по QR.*

## Личная «Главная»

Для лёгких ролей «Главная» — это персональная страница (см. [01-layout-i-menu.md](./01-layout-i-menu.md)):

| Роль | Что на «Главной» |
|------|------------------|
| **STUDENT** | Ближайшее занятие, мой пропуск, кнопка «Отметиться по QR», ссылки на расписание/предметы |
| **INSTRUCTOR** | Ближайшее занятие («Вести занятие · посещаемость»), быстрые действия (расписание, регистрация входа, предметы и тарифы) |
| **SUB_TENANT** | Ближайшее бронирование, «Забронировать зал» |
| **CLEANER** | «Отметиться по QR», переход в «Обращения» |

## Пункты группы «Моё»

Состав зависит от роли:

| Пункт | Маршрут | Кому |
|-------|---------|------|
| **Моё расписание** | `/me/schedule` | STUDENT, INSTRUCTOR |
| **Проход (QR)** | `/me/scan` | STUDENT, INSTRUCTOR, CLEANER, OWNER, ADMIN |
| **Мои пропуска** | `/access` (только свои) | STUDENT, INSTRUCTOR, SUB_TENANT, CLEANER |
| **Субаренда** | `/sublease` (свои брони) | SUB_TENANT |
| **Обращения** | `/me/requests` | INSTRUCTOR, CLEANER |

## Моё расписание (`/me/schedule`)

Компактное представление занятий пользователя (фильтрация на стороне API: преподаватель видит свои слоты, ученик — свои занятия).

## Проход (QR) (`/me/scan`)

Заголовок **«Отметка на проходной»**.

- Блок **«Станционный QR»**: область камеры, кнопки **«Запустить камеру»** / **«Остановить»**. Успех — **«Вход зарегистрирован»**.
- Блок **«или личный пропуск»**: поле **«Токен QR личного пропуска»** + **«Отметиться»** (успех — «Посещение зарегистрировано»).

Типичные отказы (по `denyCode`): неверный/просроченный код, нет пропуска, нет занятия в окне, роль не поддерживается, нет членства/залов; при частых попытках — **429**.

> **CLEANER** теперь тоже может проходить по QR (роль добавлена в станционный вход и check-in). Для уборщицы проход можно настроить «по графику» (через шаблон пропуска с контролем графика и правила «График сотрудников») или без привязки.

## Обращения (`/me/requests`)

Раздел **«Обращения»** для преподавателя и уборщицы — запросы к администрации: **отгул**, **замена**, **уведомление об отсутствии** («Не смогу прийти»).

- **«Новое обращение»** → модал: **Тип**, **Дата** (опционально), **Сообщение**.
- Таблица своих обращений со статусом (**Новое / Принято / Согласовано / Отклонено**).

Администрация обрабатывает обращения в разделе **«Обращения»** группы «Люди» — см. [09-polzovateli-grafik.md](./09-polzovateli-grafik.md).
