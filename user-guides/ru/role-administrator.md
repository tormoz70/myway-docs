# Роль ADMIN (администратор организации)

Администратор ведёт **операционную** работу студии: расписание, люди, пропуска, коммуналка, контент, заявки. По умолчанию ваш доступ к **финансовым отчётам** ограничен, пока владелец явно не включит флаг `financeReportForAdmin`.

Ниже — не общие фразы, а **что именно значит каждая функция** и где читать пошагово.

---

## 1. Расписание и помещения — что это в MyWay

### 1.1. «Ведение расписания»

**Расписание** — это календарная сетка загрузки **залов** по времени. Вы планируете:

- **Занятия** (уроки) — с предметом, преподавателями и учениками;
- **Аренду**, **обслуживание**, **блокировки** — чтобы коллеги не поставили урок на занятый или закрытый слот.

Подробный разбор интерфейса (таблица времён, цвета блоков, месячный вид, модальное окно, повторения, посещаемость, удаление):

**→ см. полную главу [02-raspisanie-podrobno.md](./02-raspisanie-podrobno.md).**

Кратко по правам администратора:

| Действие | ADMIN |
|----------|--------|
| Создать / редактировать запись | да |
| Заполнить **«Факт посещаемости»** на занятии | да |
| **Удалить** запись (**«Удалить»** в модалке) | да |

### 1.2. «Помещения» (= раздел **«Залы»**)

Это справочник залов с типом, площадью, ставкой за час и цветом в календаре. Без залов расписание не на что привязать.

**→ см. [04-zaly.md](./04-zaly.md).**

## 2. Пропуска и вход посетителей

Вы настраиваете **шаблоны** (разовый / период / пакет посещений), **выдаёте** пропуска пользователям, показываете **станционный QR** на ресепшене и смотрите **журнал**.

**→ см. [05-propusk.md](./05-propusk.md).**

Практический сценарий: новый ученик купил абонемент → создали шаблон (если ещё нет) → **«Выдать пропуск»** → ученик отмечается в **«Личном кабинете»** по камере.
