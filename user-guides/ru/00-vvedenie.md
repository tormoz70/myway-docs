# Введение: роли и адреса

## Назначение MyWay

MyWay — веб-приложение для студий и школ: расписание, учёт посещений и пропусков, биллинг помещений, субаренда, финансовые отчёты (при включённом тарифе), публичная страница организации и приём заявок от преподавателей и учеников.

## Основные URL

| Назначение | Путь |
|------------|------|
| Лендинг платформы | `/myway` |
| Вход | `/myway/login` |
| Регистрация новой студии (первый владелец) | `/myway/register` |
| Приватность и экспорт ПДн | `/myway/account/privacy` |
| Панель платформы (только SUPER_ADMIN) | `/myway/platform` |
| Публичная страница студии | `/myway/<slug>` |
| Запись преподавателем | `/myway/<slug>/join/instructor` |
| Запись учеником | `/myway/<slug>/join/student` |
| Рабочий кабинет организации | `/myway/<slug>/manage/...` |

`<slug>` — короткий адрес студии (латиница), задаётся при регистрации или автоматически из названия.

## Роли пользователей в организации

| Код роли | Кто это |
|----------|---------|
| `OWNER` | Владелец студии, полные права управления и единственный, кто может включить администраторам доступ к финансовому отчёту (флаг организации). |
| `ADMIN` | Администратор: операционное управление; финансы видит только если позволил владелец и тариф. |
| `INSTRUCTOR` | Преподаватель: ведёт занятия в расписании, посещаемость, может предлагать тарифы по своим предметам. |
| `STUDENT` | Ученик: просмотр расписания, свои пропуска, отметка прохода. |
| `SUB_TENANT` | Субарендатор: бронирования в модуле субаренды в пользовательском сценарии. |
| `SUPER_ADMIN` | Сотрудник платформы: управление тенантами и подписками, не имеет доступа к финансовым данным конкретной студии. |

Дальше см. [01-layout-i-menu.md](./01-layout-i-menu.md) и главы по разделам.

## Вход в систему

![Страница входа (продакшен-стенд)](../assets/login-page.png)

1. Откройте `/myway/login`.
2. Поля формы карточки **«Вход в систему»**:
   - **Email** — рабочая почта пользователя.
   - **Пароль**.
3. Нажмите **«Войти»**.

После успешного входа:

- пользователь **SUPER_ADMIN** попадает на `/myway/platform`;
- пользователь организации — на `/myway/<slug>/manage/dashboard`, где `<slug>` соответствует его студии.

При ошибке показывается уведомление (например «Неверный пароль или имя пользователя!»).

## Регистрация новой студии

![Регистрация студии](../assets/register-page.png)

Страница `/myway/register`, заголовок карточки **«Регистрация студии»**.

Основные поля:

- **Имя** — ФИО регистрирующегося пользователя.
- **Email**, **Пароль** (не короче 8 символов).
- **Название студии**.
- **Адрес студии (URL)** — необязательный slug; под полем подсказка проверки доступности (`/myway/<slug>`).
- Обязательный чекбокс согласия с политикой и пользовательским соглашением.
- Опционально — согласие на маркетинговые письма.

Кнопка **«Зарегистрироваться»** создаёт организацию и назначает пользователя владельцем (`OWNER`).
