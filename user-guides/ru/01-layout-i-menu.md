# Интерфейс рабочей области студии

Раздел описывает экран `/myway/<slug>/manage/*` для участников организации (не SUPER_ADMIN). Для SUPER_ADMIN меню сокращено — см. [14-platforma-super-admin.md](./14-platforma-super-admin.md).

![Общий вид: меню и контент](../assets/layout-sidebar.png)  
*Рис. 1. Пример: левое меню и область контента (см. [assets/README.md](../assets/README.md)).*

## Верхняя часть

- **Слева** — кнопка сворачивания боковой панели (иконка «складывания» меню).
- Рядом выводится **название текущей организации** (как в профиле входа).
- **Справа** — аватар и имя пользователя; по клику открывается выпадающее меню быстрых переходов (организация, участники, безопасность, выход и т.д. по роли).

## Левая колонка (боковое меню)

Заголовок **MyWay** (в свёрнутом виде может отображаться **MW**).

Пункты (подписи как в интерфейсе):

1. **Главная** — дашборд организации (для обычных ролей); у SUPER_ADMIN при открытии чужого slug здесь показывается панель платформы.
2. **Предметы**
3. **Расписание**
4. **Залы**
5. **Пропуска**
6. **Биллинг**
7. **Финансы** — подменю: **Сводный оборот**, **Категории**, **Расходы**, **Начисления** (видимость зависит от роли и настроек тарифа/флагов).
8. **Субаренда**
9. **Экспорт 1С**
10. **Преподаватели**
11. **Ученики**
12. **График админов**
13. **Новости**
14. **Заявки** — при наличии необработанных заявок рядом с подписью может отображаться числовой бейдж.
15. **Личный кабинет**
16. **Настройки**
17. **Обратная связь**

Клик по пункту меняет маршрут в правой части экрана.

## Страница «Главная» (дашборд студии)

Для обычных ролей организации отображается приветствие **«Добро пожаловать в …»** и краткий текст **«Панель управления вашей студией»**.

Блоки **статистики** (карточки с числами):

- **Записей сегодня**
- **Активных пропусков**
- **Неоплаченных счетов**
- **Бронирований субаренды**

(На текущей версии фронтенда значения могут быть нулевыми-заглушками — ориентируйтесь на фактическое наполнение вашей сборки.)

Карточка **«Быстрые действия»** содержит кнопки:

- **«Новое занятие»** → переход в **«Расписание»**
- **«Выдать пропуск»** → раздел **«Пропуска»**
- **«Регистрация входа»** → **«Пропуска»** с параметром вкладки check-in (`?tab=checkin` в URL)
- **«Создать бронирование»** → **«Субаренда»**

У **владельца** дополнительно блок **«Связь с поддержкой»**: поля **«Тема»** и **«Сообщение»** для письма команде платформы (SUPER_ADMIN).

## Область контента

Белая карточка с отступами: здесь отображается страница выбранного раздела (таблицы, формы, графики).

## Важно про права и меню

Пункт меню может быть **виден** пользователю, но отдельная операция на сервере (сохранение, удаление) для его роли может быть **запрещена**. В таком случае появится сообщение об ошибке. Детали по ролям — в файлах `role-*.md`.
