# Субаренда и экспорт в 1С

## Субаренда

Заголовок страницы **«Субаренда»**. Назначение — планировать **бронирования** помещений субарендаторами и инструкторами (в пользовательских сценариях), выставлять **счета субаренды** и фиксировать оплату.

![Доступность и бронирования](../assets/sublease-workspace.png)

### Вкладка «Доступность и бронирования»

Верхняя панель инструментов:

- **Выбор зала** (Select помещений).
- Стрелки **предыдущий / следующий день** вокруг **DatePicker** — быстрый переход по календарю.
- Кнопка **«Новое бронирование»** — открывает модал **«Новое бронирование»** (или **«Редактировать бронирование»**).

Таблица **«Доступность и бронирования»** строками объединяет:

- слоты доступности (потенциальное время);
- фактические бронирования с колонками **Зал**, **Ставка**, **Арендатор**, **Интервал**, **Сумма**, **Статус**.

Статусы бронирований (теги цветов):

- **Черновик** (`PENDING`)
- **Подтверждено** (`CONFIRMED`)
- **Отменено** (`CANCELLED`)
- **Завершено** (`COMPLETED`)

Действия в строке (иконки с подсказками):

- редактирование;
- отмена / восстановление;
- снятие подтверждения;
- **«Выставить счёт»** для подтверждённых бронирований.

### Вкладка «Счета субаренды»

Таблица счетов: номер, связанные бронирования (ссылка **«Открыть»**), сумма, статус (**Черновик**, **Отправлен**, **Оплачен**, **Просрочен**), даты выставления и оплаты. Для неоплаченных счетов есть действие **«Внести оплату»** → модал с суммой и датой.

### Модал оплаты

Заголовок **«Оплата счета №…»**, кнопка **«Сохранить оплату»**.

## Экспорт в 1С

Страница **«Экспорт в 1С»**. С релиза 0.3.x доступна из раздела **«Финансы» → «Экспорт 1С»** (путь `/go/<slug>/manage/finance/export`); старая ссылка `/manage/export` перенаправляется сюда автоматически.

![Экспорт 1С](../assets/export-1c.png)

Элементы:

1. **Период** — **RangePicker** `ДД.ММ.ГГГГ` для начала и конца выгрузки.
2. Блок **«Данные для экспорта»** — три независимых чекбокса:
   - **Контрагенты**
   - **Счета**
   - **Акты**  
   Если все сняты, показывается предупреждение **«Выберите хотя бы один тип данных для экспорта»**, кнопка экспорта неактивна.
3. Карточка **«Будет экспортировано»** — текстовая сводка выбора.
4. Кнопка **«Экспортировать»** — запрос к API; при успехе загрузка ZIP `export_1c_YYYY-MM-DD_YYYY-MM-DD.zip`.

Доступно **OWNER** и **ADMIN**.
